# Scale Tickets

The Scale Tickets skill lets you log, track, query, and export grain delivery records directly from Discord. It supports three entry methods -- snap a photo of the ticket, dictate the details by voice or text, or walk through a guided entry flow. All data is stored securely in your farm's cloud database.

## How it works

Scale ticket data is stored securely in your farm's cloud database as part of the Pantheon platform. When you enable the Scale Tickets skill, your assistant can immediately start recording and querying delivery records. No additional setup is required.

<Callout variant="success" title="Automatic database setup">
Your scale ticket database is provisioned automatically when you enable the Scale Tickets skill. No setup required on your end.
</Callout>

Each ticket record can include:

| Field | Description | Example |
|---|---|---|
| Date | Delivery date | 10/15/2025 |
| Elevator | Where the grain was delivered | CHS Fargo |
| Crop | Commodity type | Corn |
| Gross weight | Weight before unloading (lbs) | 82,340 |
| Tare weight | Truck weight after unloading (lbs) | 34,120 |
| Net weight | Grain weight (gross - tare) (lbs) | 48,220 |
| Moisture % | Moisture reading at delivery | 15.2% |
| Test weight | Weight per bushel (lb/bu) | 56.4 |
| Dockage % | Percentage docked for foreign material | 0.5% |
| Price per bushel | Cash price received | $4.52 |
| Grade | Grain grade designation | #2 Yellow |
| Truck number | Which truck made the delivery | Truck 3 |
| Load number | Sequential load identifier | 47 |
| Field name | Which field the grain came from | North 80 |
| Notes | Any additional notes | Slight dock for moisture |

## Three entry methods

### 1. Photo OCR

Post a photo of your paper scale ticket in the Discord channel. The assistant uses AI vision to read the ticket and extract all visible fields.

**How it works:**

1. Take a photo of the scale ticket with your phone.
2. Post the image in Discord.
3. The assistant reads the image and presents the extracted data for confirmation.
4. Review the summary. If anything is wrong, tell the assistant what to change.
5. Reply "save" to record the ticket.

```
Scale Ticket (from photo)
Date: 10/15/2025
Elevator: CHS Fargo
Crop: Corn
Gross: 82,340 lbs
Tare: 34,120 lbs
Net: 48,220 lbs (861.1 bu)
Moisture: 15.2%
Test Weight: 56.4 lb/bu
Price: $4.52/bu

Does this look correct? Reply "save" to record, or tell me what to change.
```

<Callout variant="tip" title="Tips for better photo OCR">
Lay the ticket flat on a dark surface. Make sure the text is in focus and well-lit. The assistant can read most printed and handwritten tickets, but legibility matters. If a field is unclear, the assistant will tell you which ones it could not read and ask you to fill in the gaps.
</Callout>

### 2. Voice / unstructured text

Type or dictate the ticket details naturally. The assistant parses your message, extracts the fields, and confirms before saving. This is the fastest method when you are in the truck and want to log a load quickly.

**Example inputs:**

```
47,672 pounds of corn from the home field, moisture 17.6%, taking it to CHS
```

```
Got a load of beans, net 48k, 13.2 moisture, CHS
```

```
Delivered 862 bushels of wheat to ADM today, $5.80
```

The assistant uses these parsing rules to interpret your message:

| Pattern | Interpretation |
|---|---|
| "k" or "K" after a number | Thousands (48k = 48,000 lbs) |
| "beans" | Soybeans |
| "durum" | Durum Wheat |
| "spring wheat" or "HRS" | Spring Wheat |
| Numbers ~40,000--90,000 | Likely net weight in pounds |
| Numbers ~400--1,800 | Likely bushels |
| Numbers 10--25 with decimal | Likely moisture percentage |
| Numbers 45--65 with decimal | Likely test weight (lb/bu) |

If no date is mentioned, today's date is used. The assistant converts between pounds and bushels automatically using the standard bushel weights table.

### 3. Multi-step structured entry

Say "new ticket", "log a ticket", or "add scale ticket" to start a guided entry flow. The assistant walks you through four groups of fields, one at a time.

**Group 1: Basics**

- **Date** -- Defaults to today if not specified
- **Elevator** -- Suggests your configured elevators
- **Crop** -- Suggests your configured crops

**Group 2: Weights**

- **Gross weight** -- Weight before unloading
- **Tare weight** -- Truck weight after unloading
- **Net weight** -- Auto-calculated (gross minus tare). You can also provide net weight directly.
- The assistant asks "pounds or bushels?" if the unit is unclear.

**Group 3: Quality** (skippable -- say "skip" or "no quality data")

- **Moisture %**
- **Test weight** (lb/bu)
- **Dockage %**

**Group 4: Details** (skippable)

- **Price per bushel**
- **Grade**
- **Truck number**
- **Load number**
- **Field name**
- **Notes**

After all groups, the assistant presents the complete ticket for confirmation before saving.

<Callout variant="info" title="Configurable fields">
Your admin can configure which fields are visible and which are required using the `visible_fields` and `required_fields` settings. If a field is not in your visible fields list, the assistant will not ask for it during structured entry -- but will still accept it if provided via photo OCR or voice.
</Callout>

## Bushel weight table

The assistant uses these standard bushel weights for all pound-to-bushel conversions:

| Crop | Lbs per bushel |
|---|---|
| Corn | 56 |
| Soybeans | 60 |
| Spring Wheat | 60 |
| Winter Wheat | 60 |
| Durum | 60 |
| Barley | 48 |
| Sunflowers | 24 |
| Canola | 50 |
| Dry Beans | 60 |
| Flax | 56 |

When displaying totals, the assistant always shows both pounds and bushels. The net bushels formula is:

```
Net Bushels = Net Weight (lbs) / Lbs per Bushel
```

For example, 48,220 lbs of corn: `48,220 / 56 = 861.1 bushels`.

## Querying your tickets

Once you have tickets logged, you can ask questions about your delivery history. Here are the supported query patterns:

### By date

```
Show today's tickets
```

```
Show tickets from last Tuesday
```

```
How many loads this week?
```

### By crop

```
Total bushels of corn
```

```
Average moisture for soybeans
```

```
How many loads of wheat so far?
```

### By elevator

```
Total by elevator
```

```
How many bushels went to CHS?
```

### Summary reports

```
Total by crop
```

```
Total by elevator
```

```
Season summary
```

### Exports

```
Export tickets
```

This generates an `.xlsx` spreadsheet containing all your scale ticket records, sorted by date (newest first), and sends it as a Discord attachment.

## Editing and deleting tickets

You can modify or remove tickets after they are saved.

| Action | What to say | What happens |
|---|---|---|
| Edit a ticket | "Edit ticket #5" | Shows ticket #5, asks what to change |
| Update a field | "Update moisture on last ticket to 15.8" | Updates the specific field on the most recent ticket |
| Delete a ticket | "Delete ticket #5" | Shows ticket details and asks for confirmation before removing |

<Callout variant="warning" title="Deletions are permanent">
When you confirm a deletion, the ticket is removed from the database. There is no undo. The assistant always shows you the full ticket details and asks for explicit confirmation before deleting.
</Callout>
