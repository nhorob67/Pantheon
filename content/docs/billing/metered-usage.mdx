## How metering works

Every interaction with your AI assistant consumes tokens — the unit of measurement for AI model usage. FarmClaw tracks token usage daily and bills it as a separate line item on top of your $40/month subscription.

### What are tokens?

Tokens are the building blocks of AI text processing. Roughly:

- **1 token** ≈ 4 characters of English text
- **100 tokens** ≈ 75 words
- A typical question and answer uses **500–2,000 tokens**

Both the message you send and the response your assistant generates count toward token usage.

## Usage by task

Different tasks consume different amounts of tokens:

| Task | Typical Token Usage |
|------|-------------------|
| Simple question/answer | 500–1,000 |
| Grain bid lookup (single elevator) | 1,500–2,500 |
| Grain bid comparison (multiple elevators) | 2,000–4,000 |
| Weather forecast | 1,000–2,000 |
| Morning weather briefing (cron) | 1,500–3,000 |
| Scale ticket — text entry | 1,000–2,000 |
| Scale ticket — photo OCR | 3,000–5,000 |
| Scale ticket — structured entry | 2,000–4,000 |
| Scale ticket query/report | 500–1,500 |

For most farm operations, metered API usage runs **$5–15/month**:

| Usage Level | Description | Estimated Cost |

## Tracking your usage

### Dashboard

Your FarmClaw dashboard shows real-time usage analytics:

- **Daily token consumption** — Bar chart showing tokens used per day
- **Cost breakdown** — Running total for the current billing period
- **Per-model breakdown** — Token usage split by AI model
- **Trend lines** — See how your usage changes over time

### Usage table

The `api_usage` table tracks daily metering per model:

- Date
- Model used
- Input tokens consumed
- Output tokens generated
- Estimated cost

Access your usage data anytime from **Settings > Usage** or the main dashboard.

## Reducing usage

If you want to keep costs down:

- **Disable unused cron jobs** — Automated briefings run daily even if you're not reading them
- **Use fewer agents** — Each agent in a multi-agent setup consumes tokens independently
- **Be specific** — Clear, direct questions use fewer tokens than vague ones
- **Use text entry** for scale tickets instead of photo OCR when possible — OCR uses significantly more tokens due to image processing
