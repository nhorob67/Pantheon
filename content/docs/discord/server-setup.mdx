## Overview

This guide walks you through creating a Discord server and connecting it to Pantheon. The entire process takes about 5 minutes.

You'll need:

- A Discord account (free at [discord.com](https://discord.com))
- Access to your Pantheon onboarding wizard or Settings page

## Step 1: Create a Discord server

If you already have a Discord server for your team, skip to Step 2.

1. Open Discord (desktop app or web)
2. Click the **+** button in the server list on the left sidebar
3. Select **Create My Own**
4. Choose **For me and my friends** (or **For a club or community** if you prefer)
5. Name your server -- something like "Acme Team" or "My Workspace"
6. Click **Create**

Your server is ready. You can customize it later with channels, roles, and permissions.

<Callout variant="tip" title="Suggested channels">
Consider creating channels that match your workflow: `#general`, `#research`, `#reports`, `#tasks`. You can bind different AI agents to each channel later.
</Callout>

## Step 2: Add Pantheon to your Discord server

Pantheon uses a single shared bot -- you don't need to create your own bot application or manage any tokens.

### During onboarding

On the Discord Connection step of the onboarding wizard, click **Add Pantheon to Discord**. This opens a standard Discord OAuth authorization page where you select your server and grant the bot the permissions it needs.

Alternatively, you can paste your **Discord Server ID** directly if you prefer.

### After onboarding

If you need to connect a different server later:

1. Go to **Settings > Discord** in your Pantheon dashboard
2. Click **Add Pantheon to Discord** or enter your Server ID
3. Click **Save**

### How to find your Server ID

If you're pasting a Server ID manually:

1. Open Discord and right-click your server name
2. Click **Copy Server ID** (you may need to enable Developer Mode in Discord settings first: User Settings > Advanced > Developer Mode)
3. Paste the ID into Pantheon

## Bot permissions

The Pantheon bot requests these permissions when you add it to your server:

| Permission | Purpose |
|------------|---------|
| Send Messages | Respond to your questions and post updates |
| Read Message History | Provide context in threaded conversations |
| Embed Links | Format responses with rich embeds (tables, cards) |
| Attach Files | Send spreadsheets, charts, and PDFs |
| Read Messages / View Channels | See messages in the channels it's assigned to |

These are the minimum permissions needed. Pantheon does not request administrator access or any permissions beyond what's listed above.

<Callout variant="success" title="You're connected">
Once you complete onboarding, your agent is ready. The Pantheon bot will appear online in your Discord server. Send it a message to verify everything is working.
</Callout>

## Troubleshooting

### Bot shows offline

- Verify your workspace is active on the Pantheon dashboard
- Check that the Pantheon bot hasn't been **kicked** from the server -- re-add it from Settings > Discord

### Bot doesn't respond to messages

- Make sure the bot has **Read Messages/View Channels** and **Send Messages** permissions in the channel you're testing
- If using channel bindings, make sure the channel you're in is bound to an agent or is handled by the default agent
- Try messaging the bot in a DM to rule out channel permission issues

### Can't add the bot to my server

- You need the **Manage Server** permission in Discord to add bots. Ask a server admin if you don't have this permission.
- If the authorization page shows an error, try logging out of Discord in your browser and logging back in.
