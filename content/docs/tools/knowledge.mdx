# Knowledge Files

Knowledge files let you upload documents that your assistant can reference when answering questions. Upload farm manuals, crop plans, insurance documents, or any other material you want your assistant to know about.

## Supported formats

| Format | Extension | Notes |
|---|---|---|
| PDF | `.pdf` | Text extracted automatically (with OCR fallback for scanned documents) |
| Word Document | `.docx` | Full text extraction |
| Plain Text | `.txt` | Direct text |
| Markdown | `.md` | Supports frontmatter |

## Limits

| Limit | Value |
|---|---|
| Max file size (upload) | 10 MB |
| Max parsed content per file | 500 KB |
| Max total storage | 2 MB across all files |
| Max number of files | 20 |

## Uploading files

1. Go to **Settings > Knowledge**
2. Drag and drop a file or click to browse
3. Choose the scope: **Shared** (all agents) or a specific agent
4. Upload

FarmClaw processes the document, extracts the text, sanitizes the content, and indexes it for retrieval. Your assistant can reference the content immediately.

## Shared vs. per-agent knowledge

| Scope | Who can access | Best for |
|---|---|---|
| **Shared** | All agents in your workspace | General farm documents, SOPs, reference material |
| **Per-agent** | Only the assigned agent | Specialized documents relevant to one agent's role |

You can reassign files between shared and per-agent at any time using the dropdown on each file's card.

<Callout variant="info" title="Agent deletion preserves files">
If you delete an agent that has per-agent knowledge files, those files are automatically promoted to shared so they remain accessible.
</Callout>

## How retrieval works

When your assistant receives a question, it searches your knowledge files using a hybrid approach:

- **Semantic search** -- Finds content that is conceptually similar to the question using vector embeddings, even if different words are used
- **Keyword search** -- Matches specific terms and phrases from the question

The most relevant passages are included in the assistant's context when generating a response. Your assistant will cite the source document when using knowledge file content.

Large documents are automatically split into chunks for efficient retrieval. No configuration needed.

## Managing knowledge files

From **Settings > Knowledge**, you can:

- **View** all uploaded files with their type, size, status, and agent assignment
- **Monitor storage** -- A progress bar shows your total usage
- **Reassign scope** -- Move files between shared and per-agent
- **Delete** files you no longer need
- **Review failed uploads** -- See error messages and retry or remove

<Callout variant="tip" title="Quality over quantity">
A few well-chosen, relevant documents will give better results than uploading everything you have. Focus on documents your assistant is likely to need when answering questions.
</Callout>
