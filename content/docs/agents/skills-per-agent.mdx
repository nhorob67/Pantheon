# Skills & Scheduling

Each agent in your Pantheon workspace can be assigned custom skills and its own set of scheduled messages. This lets you create focused specialists — a Support Lead with FAQ skills, a Research Analyst with web search, or an Operations Lead with daily standup schedules.

## Custom skills

Skills are Markdown-based instructions (SKILL.md format) that teach agents how to handle specific tasks. All skills in Pantheon are custom — you create them for your specific use case.

### Creating skills

There are three ways to create skills:

1. **Skill Forge** — Write a skill from scratch using the built-in Markdown editor
2. **AI Generation** — Describe what you want and Pantheon generates the skill using AI
3. **Extension Marketplace** — Install pre-built skills from the community

### Assigning skills to agents

1. Go to **Agents** in the sidebar
2. Select the agent you want to configure
3. Under **Skills**, toggle each custom skill on or off
4. Click **Save Changes**

Each agent only has access to its assigned skills. A Research Analyst might have a "Web Search" skill, while a Support Lead has "FAQ Lookup" and "Ticket Triage" skills.

<Callout variant="tip" title="Skills are shared or per-agent">
Create a skill once, then assign it to any combination of agents. A skill like "Company Knowledge Base" might be assigned to every agent, while "Competitor Analysis" is only assigned to the Research Analyst.
</Callout>

## Schedules (automated messages)

Agents can send scheduled messages to their assigned Discord channel on a cron-based schedule.

### How schedules work

Each schedule has:

| Field | Description |
|-------|-------------|
| **Label** | A human-readable name (e.g., "Daily Summary") |
| **Cron expression** | When the schedule runs (e.g., `0 8 * * 1-5` for weekdays at 8 AM) |
| **Message template** | The prompt sent to the agent when the schedule fires |
| **Enabled** | Whether the schedule is currently active |

When a schedule fires, Pantheon sends the message template to the agent as if a user had typed it. The agent processes it using its role, goal, skills, and knowledge, then posts the response to its Discord channel.

### Built-in schedule templates

Pantheon provides generic schedule templates to get you started:

| Template | Default schedule | What it does |
|----------|-----------------|-------------|
| **Daily Summary** | Weekdays at 8:00 AM | Agent reviews the day's priorities and posts a summary |
| **Weekly Report** | Mondays at 9:00 AM | Agent compiles a weekly overview |
| **Morning Standup** | Daily at 7:30 AM | Agent posts a structured standup prompt |

These are starting points — you can customize the schedule, message template, and everything else.

### Creating custom schedules

1. Go to **Agents** in the sidebar
2. Select an agent
3. Under **Schedules**, click **+ Add Schedule**
4. Set the label, cron expression, and message template
5. Save

Schedules run in the agent's timezone (inherited from the team timezone set during onboarding).

<Callout variant="info" title="Schedules run in the agent's channel">
Scheduled messages are posted to the agent's bound Discord channel. If the agent has no channel binding, the schedule posts to the default channel.
</Callout>

## Channel bindings

Each agent can be bound to a specific Discord channel:

- **Bound channels** — Only the assigned agent responds in that channel
- **Unbound channels** — The **default agent** handles messages
- **DMs** — The default agent always handles direct messages

This lets you create dedicated channels: `#support-tickets` for the Support Lead, `#research` for the Research Analyst, etc.

<Callout variant="tip" title="One agent must be the default">
Every workspace must have exactly one default agent. It handles DMs and any channel not bound to a specific agent.
</Callout>

## Example: Setting up a three-agent team

**1. Create your custom skills.**

Go to Settings > Skills and create the skills your agents will need (e.g., "FAQ Lookup", "Ticket Triage", "Web Search").

**2. Create Agent 1: General Assistant (default).**

- Role: General operations assistant
- Goal: Keep the team organized and route work to the right place
- Skills: FAQ Lookup
- Channel: (none — handles unbound channels and DMs)
- Default: Yes

**3. Create Agent 2: Support Lead.**

- Role: Customer support specialist
- Goal: Resolve customer questions quickly and accurately
- Skills: FAQ Lookup, Ticket Triage
- Schedule: Daily Summary at 8:00 AM
- Channel: `#support-tickets`
- Default: No

**4. Create Agent 3: Research Analyst.**

- Role: Research analyst
- Goal: Investigate questions and provide clear recommendations
- Skills: Web Search
- Schedule: Weekly Report on Mondays
- Channel: `#research`
- Default: No

**5. Save and test.**

Each agent responds in their designated channel. The General Assistant handles everything else and delegates specialist questions when appropriate.
