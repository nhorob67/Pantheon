## Overview

Every Pantheon workspace has at least one AI agent — your team's first worker. Agents are fully custom: you define their role, goal, backstory, and autonomy level. There are no locked-in presets. You build the team you need.

Pantheon supports **multiple agents** per workspace. Each agent can have its own identity, skills, and Discord channel assignment. Think of it as building a small team of AI specialists for your operation.

## Single agent vs. multi-agent

### Single agent (the default)

When you complete onboarding, you create one agent with a role and goal you define. It handles every channel in your Discord server plus DMs.

This is the simplest setup and works well for many teams. One agent that does everything.

### Multi-agent setup

As your team grows or you want more focused responses, add specialist agents:

- A **Support Lead** in `#support-tickets` that resolves customer questions quickly
- A **Research Analyst** in `#research` that investigates questions and synthesizes findings
- An **Operations Lead** in `#operations` that tracks work and keeps execution moving
- A **Comms Manager** in `#comms` that handles email follow-ups and scheduling

Each specialist is tuned for its area. They know their boundaries and can delegate to other agents when a question falls outside their expertise.

<Callout variant="tip" title="Start with one agent">
You can always add more agents later. Start with a single general-purpose agent and see how it handles your day-to-day questions. When you want more focused responses on a specific topic, add a specialist.
</Callout>

## Agent identity: Role, Goal, and Backstory

Every agent is defined by three fields inspired by the CrewAI pattern:

| Field | Purpose | Example |
|-------|---------|---------|
| **Role** | What the agent is — its job title or function | "Customer support specialist" |
| **Goal** | What the agent aims to accomplish | "Resolve customer questions quickly and accurately" |
| **Backstory** | Personality, tone, and constraints (optional) | "You are friendly but concise. Always cite sources." |

These fields drive the agent's system prompt and shape how it responds. The more specific you are, the better the agent performs within its domain.

## Autonomy levels

Each agent has an autonomy level that controls how independently it acts:

| Level | Label | Behavior |
|-------|-------|----------|
| L1 | **Assisted** | Asks before taking any action. Presents options and waits for approval. |
| L2 | **Copilot** | Suggests actions and explains reasoning. Executes read-only operations automatically. |
| L3 | **Autopilot** | Acts independently based on its goal. Only asks when genuinely ambiguous. |

## Skills and schedules

Agents can be given **custom skills** — Markdown-based instructions that teach them how to handle specific tasks. Create skills from scratch or generate them with AI via the Skill Forge.

Agents can also have **custom schedules** — cron-based scheduled messages that post to their assigned Discord channel automatically. For example, a daily summary at 8 AM or a weekly report every Monday.

## Inter-agent communication

When you have multiple agents, they can delegate tasks to each other using built-in communication tools (`agents_list` and `sessions_send`). If someone asks the general assistant a detailed research question, it can hand off to the Research Analyst.

Users see a seamless experience — they ask a question in any channel, and the right agent handles it.
