## Overview

Pantheon workspaces can run multiple agents, each with their own role, goal, skills, and Discord channel. Agents can delegate tasks to each other, creating a team of specialists that collaborates seamlessly.

## Building a multi-agent team

Every team starts with one agent created during onboarding. Add more agents from the Agents page as your needs grow.

### Example: Customer Support Team

| Agent | Role | Channel | Autonomy |
|-------|------|---------|----------|
| **General Assistant** | General operations assistant | All channels + DMs | Copilot |
| **Support Lead** | Customer support specialist | `#support-tickets` | Copilot |
| **Research Analyst** | Research analyst | `#research` | Assisted |

### Example: Operations Team

| Agent | Role | Channel | Autonomy |
|-------|------|---------|----------|
| **Executive Assistant** | General operations assistant | All channels + DMs | Copilot |
| **Operations Lead** | Operations lead | `#operations` | Autopilot |
| **Research Analyst** | Research analyst | `#research` | Assisted |
| **Comms Manager** | Communications manager | `#comms` | Copilot |

These are examples — you define the roles, goals, and channel assignments. There are no fixed configurations.

<Callout variant="tip" title="Start small">
Start with one or two agents and add specialists as you identify specific needs. 2-4 agents is the practical sweet spot for most teams.
</Callout>

## How delegation works

When agents have delegation enabled, they can route questions to the right specialist automatically.

### The tools

Each agent with delegation enabled has access to two communication tools:

| Tool | Purpose |
|------|---------|
| `agents_list` | Lists all other agents, their roles, and channel bindings |
| `sessions_send` | Sends a message to another agent and receives a response |

### Example flow

1. A user asks the General Assistant in `#general`: "Can you research our competitor's pricing?"
2. The General Assistant checks `agents_list` and sees the Research Analyst is available
3. It tells the user: "Let me hand this off to our research specialist..."
4. It uses `sessions_send` to pass the question to the Research Analyst
5. The Research Analyst processes the request and returns findings
6. The General Assistant relays the response in `#general`

The user gets a specialist-quality answer without leaving their channel.

### Delegation settings

Each agent has two delegation toggles:

| Setting | What it controls |
|---------|-----------------|
| **Can delegate** | This agent can send tasks to other agents |
| **Can receive delegation** | This agent accepts tasks from other agents |

Delegation is driven by the agent's role and goal. Agents learn which questions to handle themselves and which to pass along based on these fields.

<Callout variant="info" title="Delegation is automatic but optional">
If the target agent doesn't exist or delegation is disabled, the originating agent handles the question itself. You don't need to configure routing rules — agents figure it out from their role definitions.
</Callout>

## Channel routing

Each agent can be bound to a specific Discord channel. The **default agent** handles all unbound channels and DMs.

| Configuration | What happens |
|--------------|-------------|
| Agent bound to `#support-tickets` | Only responds in that channel |
| Default agent | Handles `#general`, DMs, and any other unbound channels |
| No channel binding | Agent only responds to delegated tasks |

Every workspace must have exactly one default agent.

## Creating agents

1. Go to **Agents** in the sidebar
2. Click **+ New Agent**
3. Choose to start from a **template** or **from scratch**
4. Define the role, goal, backstory, and autonomy level
5. Optionally bind to a Discord channel
6. Add custom skills and schedules
7. Save

<Callout variant="info" title="Templates are starting points">
Agent templates pre-fill role, goal, and backstory to help you get started quickly. You can customize every field after selecting a template.
</Callout>

## Practical tips

- **One default agent is required.** It catches everything that doesn't have a dedicated specialist.
- **Use clear, non-overlapping roles.** If two agents have similar goals, they may compete on the same questions.
- **Enable delegation on the default agent.** It should be able to route questions to specialists.
- **Specialists can be receive-only.** Set "Can delegate" to off for focused agents that shouldn't redirect questions elsewhere.
- **Test with the Preview feature.** Send test messages to see how routing and delegation work before going live.
