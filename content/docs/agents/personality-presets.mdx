## The Role / Goal / Backstory Pattern

Every Pantheon agent is defined by three core identity fields. This pattern gives you precise control over how each agent behaves without relying on fixed presets.

### Role

**What is this agent?**

A short description of the agent's function — think of it as a job title.

**Examples:**
- "Customer support specialist"
- "Research analyst"
- "Operations lead"
- "Content editor and proofreader"

Keep it under 200 characters. Be specific — "Sales development representative" is better than "Sales agent."

### Goal

**What should this agent accomplish?**

The agent's primary objective. If this agent does its job well, what outcome do we see?

**Examples:**
- "Resolve customer questions quickly and accurately using our knowledge base"
- "Track open work, keep execution moving, and make sure nothing important stalls"
- "Investigate questions, synthesize findings, and provide clear recommendations with caveats"

Focus on outcomes, not process. A good goal helps the agent prioritize when it has multiple options.

### Backstory (optional)

**Personality, tone, and constraints.**

Use it to define communication style, set boundaries, or provide important context.

**Examples:**
- "You are calm, structured, and concise. Summarize decisions clearly, surface blockers early, and avoid unnecessary complexity."
- "You are warm and helpful but never make up information. Always cite your sources."
- "You think in priorities, dependencies, and deadlines. You are direct and focused on turning ambiguity into next actions."

Include constraints ("never", "always") for important boundaries. Include tone guidance ("friendly", "formal", "direct"). Up to 2000 characters.

## Autonomy Levels

Every agent has an autonomy level that controls how independently it operates.

### L1 — Assisted

Always asks before taking any action. Presents options and waits for explicit approval.

**Best for:** High-stakes decisions, new agents you're still calibrating, situations where human oversight is critical.

### L2 — Copilot (default)

Suggests actions and explains reasoning. Executes read-only operations automatically but asks before write operations. Proactively surfaces relevant information.

**Best for:** Most agents. Good balance of autonomy and oversight.

### L3 — Autopilot

Takes actions independently based on its goal. Only asks when genuinely ambiguous or high-stakes. Reports results after execution.

**Best for:** Well-defined operational tasks, agents with clear boundaries, high-volume routine work.

## Delegation

Agents can collaborate through delegation:

| Setting | Description |
|---------|-------------|
| **Can delegate** | This agent can hand off tasks to other agents |
| **Can receive delegation** | This agent accepts tasks from other agents |

When delegation is enabled, agents use `agents_list` to discover teammates and `sessions_send` to pass messages. Users see a seamless experience.

## Agent Templates

When creating a new agent, you can start from a **template** that pre-fills the role, goal, backstory, and autonomy level. Templates are starting points — you can customize every field after selecting one.

Available templates include:
- **General Assistant** — Flexible first agent for triage, coordination, and everyday requests (Copilot)
- **Operations Lead** — Execution tracking, SOP follow-through, and issue escalation (Autopilot)
- **Research Analyst** — Structured research, synthesis, and knowledge-heavy questions (Assisted)
- **Customer Support** — Inbound questions, resolution handoff, and service quality (Copilot)

You can also create agents entirely from scratch without a template.

## Tips for Effective Agent Design

1. **Start specific, broaden later.** A narrow role with a clear goal performs better than a vague one.
2. **Use backstory for constraints.** If there are things the agent should never do, put them in the backstory.
3. **Match autonomy to risk.** Use L3 Autopilot for low-risk routine tasks; L1 Assisted for anything with consequences.
4. **Test with real questions.** Use the Preview feature to test how your agent responds before deploying.

## What all agents share

Regardless of how you configure them, every Pantheon agent includes:

- **Team context** — Your team name, team goal, and timezone are embedded in the system prompt
- **Security boundaries** — Protection against prompt injection, credential leaks, and unauthorized modifications
- **File creation** — All agents can generate spreadsheets, CSVs, PDFs, charts, and other files as Discord attachments
- **Inter-agent communication** — All agents can discover teammates and delegate questions
- **Knowledge access** — Agents can reference uploaded knowledge files (shared or per-agent)
