# SQLite Server

The SQLite MCP server gives your AI assistant access to a local SQLite database, enabling it to store, query, and manage structured data. It is the backbone of the [Scale Tickets](/docs/skills/scale-tickets) skill and can also be used for any custom data storage you need.

## Overview

| Field | Value |
|---|---|
| **Package** | `@modelcontextprotocol/server-sqlite` |
| **Command** | `npx` |
| **Arguments** | `-y @modelcontextprotocol/server-sqlite /home/node/data/farmclaw.db` |
| **Scope** | Instance (available to all agents) |
| **Database path** | `/home/node/data/farmclaw.db` |

<Callout variant="tip" title="Automatically configured with Scale Tickets">
This server is automatically added to your instance configuration when you enable the Scale Tickets skill. You do not need to add it manually if Scale Tickets is active. FarmClaw detects that the skill requires SQLite and injects the server into your config during the build process.
</Callout>

## What it enables

With the SQLite server active, your assistant can:

- **Create tables** -- Define schemas for any kind of structured data.
- **Insert records** -- Add new rows to tables, either one at a time or in bulk.
- **Query data** -- Run `SELECT` statements with filters, aggregations, sorting, and joins.
- **Update and delete records** -- Modify or remove existing data.
- **Describe schemas** -- Inspect table structures, column types, and indexes.
- **Run arbitrary SQL** -- Execute any valid SQLite statement against the database.

## Scale Tickets integration

The primary use of the SQLite server is powering the Scale Tickets skill. When you enable Scale Tickets, FarmClaw:

1. Automatically adds the SQLite MCP server if it is not already configured.
2. Points the database at `/home/node/data/farmclaw.db`.
3. Initializes the scale tickets table schema via the container's entrypoint script.
4. Configures the skill with your crop list and elevator names for bushel weight conversions.

The assistant uses the database to store every scale ticket entry -- whether submitted via photo OCR, voice dictation, or guided entry -- and can query this data to answer questions like:

> "How many bushels of corn have I delivered to CHS this month?"

> "Show me all soybeans tickets from the last two weeks sorted by date."

> "What's my total delivery weight across all elevators for the 2025 harvest?"

### Database schema

The Scale Tickets skill creates a `scale_tickets` table with fields for load date, crop, elevator, gross weight, tare weight, net weight, moisture, test weight, dockage, and more. The exact visible and required fields are configurable per-customer through the skill settings.

<Callout variant="info" title="Database persists across restarts">
The database file at `/home/node/data/farmclaw.db` is stored on a persistent volume. Your data survives container restarts, configuration rebuilds, and MCP server toggles. Disabling the SQLite server does not delete the database.
</Callout>

## Custom data storage

The SQLite server is not limited to scale tickets. Your assistant can create additional tables for any structured data you want to track:

- **Field scouting logs** -- Record observations, pest counts, or crop staging notes by field and date.
- **Equipment maintenance records** -- Track service intervals, parts, and costs.
- **Custom yield tracking** -- Store harvest data in whatever schema fits your operation.
- **Fuel and input logs** -- Record purchases, prices, and application rates.

Simply ask your assistant to create a table and start adding records:

> "Create a table called `field_scout` with columns for date, field name, crop, growth stage, observations, and action items."

The assistant will create the schema and confirm it is ready for use.

## Configuration

When added from the preset selector or auto-configured by the Scale Tickets skill, the SQLite server uses these values:

| Field | Value |
|---|---|
| `server_key` | `sqlite` |
| `display_name` | `SQLite` |
| `command` | `npx` |
| `args` | `["-y", "@modelcontextprotocol/server-sqlite", "/home/node/data/farmclaw.db"]` |
| `env_vars` | `{}` (none required) |
| `scope` | `instance` |
| `enabled` | `true` |

### Database path

The default database path is `/home/node/data/farmclaw.db`. All data is stored in a single SQLite file on a persistent volume. If you need a separate database for a different purpose, you can add a second SQLite MCP server with a different `server_key` and database path.

<Callout variant="warning" title="Changing the database path breaks Scale Tickets">
The Scale Tickets skill is hardcoded to use `/home/node/data/farmclaw.db`. If you change the database path on the default SQLite server, the skill will not be able to read or write scale ticket records. Add a separate server instead if you need a second database.
</Callout>

## Adding the SQLite server manually

If you want to add the SQLite server without enabling the Scale Tickets skill:

1. Go to **Settings > MCP Servers**.
2. Click **Add MCP Server**.
3. Select **From Preset**.
4. Click **SQLite** from the preset list.
5. Review the pre-filled fields and click **Add Server**.

The configuration rebuild takes 15--30 seconds. Once complete, your assistant can create tables and start querying data.

## Enabling and disabling

Toggle the SQLite server on or off using the switch on its card in **Settings > MCP Servers**. When disabled:

- Your assistant loses the ability to run SQL queries.
- The Scale Tickets skill will not be able to store or retrieve entries.
- The database file itself is **not deleted** and will be available again when you re-enable the server.
