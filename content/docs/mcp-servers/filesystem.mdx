# Filesystem Server

The Filesystem MCP server gives your AI agent the ability to read, write, and manage files. This is useful for saving reports, keeping notes, storing exported data, and working with files that persist across conversations.

## Overview

| Field | Value |
|---|---|
| **Package** | `@modelcontextprotocol/server-filesystem` |
| **Command** | `npx` |
| **Arguments** | `-y @modelcontextprotocol/server-filesystem` |
| **Scope** | Workspace (available to all agents) |

The Filesystem server is a custom MCP server you can add to your workspace if you need file read/write capabilities. It is not required for any built-in platform tools -- memory, knowledge files, and email all work without it.

## What it enables

With the Filesystem server active, your agent can:

- **Read files** -- Open and read the contents of any file in its workspace directory.
- **Write files** -- Create new files or overwrite existing ones. Useful for saving reports, exporting data, or generating documents.
- **List directory contents** -- Browse what files and folders exist in the workspace.
- **Create directories** -- Organize files into folders as needed.
- **Move and rename files** -- Restructure workspace content without manual intervention.

## Use cases

### Saving reports

Ask your agent to generate a summary and save it to a file:

> "Summarize this week's key metrics and save the report to `reports/weekly-summary-2025-10-13.md`."

The agent will compile the data and write it to the specified path.

### Exporting data

If you want a local copy of records or reports:

> "Export all completed tasks from October as a CSV file."

The agent writes the CSV to the workspace where it persists between conversations.

### Keeping operational notes

Your agent can maintain running notes, checklists, or logs:

> "Add a note to `meeting-notes.md` that the design review is rescheduled to Thursday."

<Callout variant="info" title="Files persist across conversations">
Files saved by the Filesystem server persist between conversations and are available any time you interact with your agent.
</Callout>

## Adding the Filesystem server

1. Go to **Settings > MCP Servers**.
2. Click **Add MCP Server**.
3. Select **From Preset**.
4. Click **Filesystem** from the preset list.
5. Review the pre-filled fields and click **Add Server**.

The server is available to your agent after saving.

## Enabling and disabling

You can toggle the Filesystem server on or off at any time using the switch on its card in **Settings > MCP Servers**. Disabling the server removes it from your agent's active tools. Your files are not deleted when the server is disabled.
