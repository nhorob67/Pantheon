# MCP Servers

Model Context Protocol (MCP) servers let you extend your AI assistant with custom tools and data access beyond the built-in farm tools. MCP servers are for advanced or custom tooling -- the core farm capabilities (scale tickets, grain bids, weather, and memory) are handled natively by the FarmClaw tenant runtime and do not require any MCP configuration.

## What is MCP?

MCP is an open protocol that lets AI assistants connect to external tool servers using a standardized interface. Instead of building every tool directly into the AI model, MCP servers expose capabilities that the assistant can call on demand. This makes it possible to integrate specialized or third-party tools without modifying the core assistant.

When your assistant needs to use a custom tool provided by an MCP server, it sends a request to that server. The server processes the request and returns the result, which the assistant then uses to answer your question.

<Callout variant="info" title="Built-in tools don't need MCP">
Scale tickets, grain bids, weather forecasts, and memory are all built into the FarmClaw platform. They work out of the box with no MCP server setup. MCP servers are only needed if you want to add custom or third-party tools.
</Callout>

## When to use MCP servers

MCP servers are useful when you need your assistant to access tools or data sources that are not covered by the built-in farm skills. Examples include:

- A custom internal API for your farm management software
- A specialized data source or database
- Third-party tool servers published by the MCP community
- Filesystem access for saving and reading custom files

## Adding a custom MCP server

MCP servers are managed through the FarmClaw dashboard.

1. Go to **Settings > MCP Servers**.
2. Click **Add MCP Server**.
3. Fill in the required fields and click **Add Server**.

You can enable, disable, edit, or delete servers at any time.

When adding a custom server, you will need to provide:

| Field | Description | Example |
|---|---|---|
| `server_key` | Unique lowercase identifier (letters, numbers, hyphens) | `my-custom-server` |
| `display_name` | Human-readable label shown in the dashboard | `My Custom Server` |
| `command` | The executable to run | `npx` |
| `args` | Arguments passed to the command (space-separated) | `-y @my-org/my-mcp-server /data` |
| `env_vars` | Optional environment variables for the server process | `{"API_KEY": "..."}` |
| `scope` | `instance` (all agents) or `agent` (specific agent only) | `instance` |
| `enabled` | Whether the server is active | `true` |

<Callout variant="tip" title="Server keys are immutable">
The `server_key` cannot be changed after creation. If you need a different key, delete the server and create a new one.
</Callout>

## Scoping

MCP servers are defined at the workspace level by default, meaning every agent in a multi-agent setup can use them. You can also scope a server to a specific agent if you want to restrict access.

Each server is identified by a unique `server_key`. The key is used internally to reference the server in the assistant's configuration.
